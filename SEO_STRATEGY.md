### 2.2 Landing Page SEO Template

Each landing page should have:

**On-Page Elements:**
```
<title>: Primary Keyword + Modifier (60 chars max)
Example: "Bigha to Acre Converter - Instant Calculation & Formula"

<meta description>: Answer + Benefit + CTA (155 chars)
Example: "Convert Bigha to Acre instantly. Get exact formula, common examples and use the free India land converter for quick plot size checks."

<h1>: Main keyword with benefit
Example: "How to Convert Bigha to Acre - Instant Converter"

<h2>: Supporting keywords
- "Bigha to Acre Conversion Formula"
- "Common Bigha Plot Sizes"
- "Why Convert Bigha to Acre?"

<h3>: Sub-topics
- "What is 1 Bigha in Acres?"
- "Example Conversions"
- "State-Specific Bigha Values"

Internal Links: 
- Link to homepage (anchor: "Main Converter")
- Link to 4 other landing pages (anchor: specific converter names)
- Link to FAQ page

Outbound Links:
- Agriculture banks (affiliate)
```

**Content Structure (500-800 words minimum):**
1. Hero section + value prop
2. Quick conversion tool (embeds converter.js)

4. Examples/reference table
5. FAQ section (5-7 questions)
6. Related links/guides

### 2.3 Schema Markup Strategy

**Implement on All Pages:**

1. **FAQPage Schema** (for featured snippets)
```json
{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [
    {
      "@type": "Question",
      "name": "How many acres is 1 Bigha?",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "1 Bigha equals 0.4-1.25 acres (varies by state)"
      }
    }
  ]
}
```

2. **SoftwareApplication Schema** (Homepage only)
```json
{
  "@context": "https://schema.org",
  "@type": "SoftwareApplication",
  "name": "India Agricultural Land Unit Converter",
  "description": "Free converter for Indian land units",
  "applicationCategory": "Utility"
}
```

3. **BreadcrumbList Schema** (on all pages)
```json
{
  "@context": "https://schema.org",
  "@type": "BreadcrumbList",
  "itemListElement": [
    {
      "@type": "ListItem",
      "position": 1,
      "name": "Home",
      "item": "https://github.io/india-agri-converter/"
    },
    {
      "@type": "ListItem",
      "position": 2,
      "name": "Bigha to Acre",
      "item": "https://github.io/india-agri-converter/guides/bigha-to-acre/"
    }
  ]
}
```

4. **Article Schema** (Landing pages)
```json
{
  "@context": "https://schema.org",
  "@type": "Article",
  "headline": "Bigha to Acre Converter",
  "author": {
    "@type": "Organization",
    "name": "India Agri Converter"
```

## Part 4: Technical SEO

### 4.1 URL Structure

**Best Practice:**
- Homepage: `/index.html` (or domain root)
- Guides: `/guides/[conversion-type]/index.html`
- Clean, descriptive, keyword-rich

**Examples:**
```
✅ /guides/bigha-to-acre/
✅ /guides/bigha-to-hectare/
❌ /p/123/ (not descriptive)
❌ /guide/converter.php (not clean)
```

---

### 4.2 Internal Linking Strategy

**Homepage Links To:**
- All 5 Tier 1 landing pages
- FAQ page (if separate)
- About/Privacy pages

**Anchor Text Examples:**
- ✅ "Bigha to Acre Converter" (keyword-rich)
- ✅ "Convert Bigha to Hectare" (transactional intent)
- ❌ "Click here" (no keyword value)
- ❌ "Read more" (no context)

**Linking Pattern:**
- Homepage → 5 landing pages (1 link each)
- Each landing page → Homepage (1 link)
- Each landing page → 4 other landing pages (related conversions)
- Result: Each page has 5-6 internal links

---

### 4.3 Mobile Optimization

**Checklist:**
- ✅ Viewport meta tag (already in place)
- ✅ Font sizes > 12px
- ✅ Touch-friendly buttons (> 44px)
- ✅ No horizontal scroll
- ✅ Form inputs optimized for mobile

**Testing:**
- Google Mobile-Friendly Test
- Chrome DevTools mobile view
- Real device testing

---

### 4.4 Page Speed Optimization

**Targets:**
- First Contentful Paint (FCP): < 1.8 seconds
- Largest Contentful Paint (LCP): < 2.5 seconds
- Cumulative Layout Shift (CLS): < 0.1

**Optimization Tactics:**
1. Minimize CSS/JS (already done: ~30KB total)
2. Lazy-load images
3. Cache strategy
4. Compress assets
5. CDN for static files

---

### 4.5 Core Web Vitals

**LCP (Largest Contentful Paint):**
- Target: < 2.5 seconds
- Currently: Excellent (converter.js is lightweight)

**FID (First Input Delay):**
- Target: < 100ms
- Currently: Excellent (vanilla JS, no framework)

**CLS (Cumulative Layout Shift):**
- Target: < 0.1
- Risk: None (fixed layout)
